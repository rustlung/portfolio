# Архитектура проекта

## 1. Общая идея

Сайт-портфолио представляет собой серверное веб-приложение на FastAPI с рендерингом HTML через Jinja2.

Особенности:

- публичная часть сайта для посетителей;
- административная панель для управления контентом;
- хранение данных в SQLite на старте;
- архитектура с возможностью дальнейшего перехода на PostgreSQL без полной переработки проекта.

---

## 2. Основные сущности

### Profile

Хранит основную информацию обо мне:

- full_name
- title
- subtitle
- about_text
- photo_path

### Contact

Хранит контакты для отображения на сайте:

- type
- label
- value
- url
- sort_order

Примеры type:

- email
- telegram
- github
- linkedin

### SkillCategory

Категория навыков:

- name
- sort_order

Примеры:

- Backend
- AI
- Infrastructure
- Integrations

### Skill

Отдельный навык:

- name
- category_id
- sort_order

### Project

Карточка проекта:

- title
- slug
- short_description
- full_description
- technologies
- github_url
- demo_url
- image_path
- is_featured
- sort_order

---

## 3. Связи между сущностями

- Один `Profile` связан со многими `Contact`
- Одна `SkillCategory` содержит много `Skill`
- Проекты (`Project`) существуют независимо от других сущностей

---

## 4. Публичные страницы

### `/`

Главная страница:

- hero-блок;
- блок "О себе";
- контакты;
- категории навыков;
- топ-3 избранных проекта.

### `/projects`

Страница всех проектов:

- список всех проектов;
- краткая информация;
- технологии;
- ссылка на подробную страницу.

### `/projects/{slug}`

Детальная страница проекта:

- название;
- изображение;
- полное описание;
- функциональность;
- технологии;
- ссылки на GitHub и demo.

---

## 5. Административная часть

### `/admin/login`

Страница входа администратора.

### `/admin`

Главная страница админки:

- быстрый переход к управлению контентом.

### Управление контентом

Нужны CRUD-разделы для:

- профиля;
- контактов;
- категорий навыков;
- навыков;
- проектов.

---

## 7. Принципы архитектуры

- Простая и читаемая структура
- Разделение публичных и административных маршрутов
- SQLAlchemy для работы с БД
- Контент не зашивается в шаблоны, а загружается из базы
- Один универсальный шаблон для страницы проекта
- Подготовка к масштабированию без усложнения MVP

---

## 8. MVP-границы первой версии

В первую версию входят:

- публичная главная страница;
- список проектов;
- страницы проектов;
- авторизация администратора;
- CRUD для проектов, навыков, контактов и блока "О себе";
- загрузка изображений для проектов.

Не входят в первую версию:

- регистрация пользователей;
- роли и права доступа;
- сложная аналитика;
- SPA/frontend-framework архитектура.
