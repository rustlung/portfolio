# Portfolio Website — Контекст проекта

## Цель проекта

Разработать персональный сайт-портфолио для Вадима Сигаева (Python Backend Developer).

Сайт должен решать несколько задач:

1. Портфолио для поиска работы.
2. Презентация проектов для HR и технических специалистов.
3. Демонстрация навыков backend-разработки.
4. Централизованный каталог выполненных проектов.
5. Основа для дальнейшего развития портфолио без изменения исходного кода.

Сам сайт также должен стать полноценным проектом в портфолио.

---

# Основная концепция

Сайт НЕ должен быть статическим лендингом.

Это должно быть небольшое CMS-подобное приложение, где весь контент хранится в базе данных и управляется через административную панель.

Проекты, навыки, контакты и информация обо мне должны редактироваться без изменения кода.

# Философия дизайна

Минимализм.

Без перегруженных анимаций.

Без шкал навыков.

Без процентов владения технологиями.

Без ярких градиентов и визуального шума.

Основной акцент на:

- читаемость;
- аккуратную верстку;
- содержание проектов;
- качественные мокапы и скриншоты.

Сайт должен выглядеть профессионально и соответствовать профилю backend-разработчика.

Отображает список всех проектов.

## Страница проекта

Маршрут:

/projects/{slug}

Примеры:

/projects/documind

/projects/ai-client-report-generator

/projects/openai-telegram-bot

Содержимое страницы:

- главный мокап;
- описание проекта;
- реализованный функционал;
- используемые технологии;
- практическое применение;
- ссылка на GitHub;
- ссылка на демо (если есть).

Важно:

Для всех проектов используется один шаблон.

Контент подгружается из базы данных.

Отдельные HTML-страницы под каждый проект не создаются.

# Административная панель

Маршрут:

Требуется авторизация.

Пользователь только один — владелец сайта.

Регистрация пользователей не требуется.

Вход по паролю.

# База данных

На старте используется SQLite.

Причины:

- простота;
- небольшой объем данных;
- удобство деплоя.

В будущем возможна миграция на PostgreSQL.

Архитектура должна быть подготовлена к такой миграции.

Для работы с БД использовать SQLAlchemy.

# Ограничения проекта

Сайт должен демонстрировать именно backend-компетенции. Но при этом выглядеть современно, стильно

Не использовать:

- React;
- Next.js;
- Vue;
- SPA-архитектуру.

Предпочтительный вариант — серверный рендеринг через Jinja2.

# План разработки

1. Спроектировать структуру базы данных.
2. Реализовать модели SQLAlchemy.
3. Реализовать авторизацию администратора.
4. Реализовать CRUD для проектов.
5. Реализовать публичные страницы сайта.
6. Реализовать загрузку изображений.
7. Подготовить Docker-конфигурацию.
8. Выполнить деплой через Nginx.

Архитектура должна оставаться простой, поддерживаемой и соответствовать уровню сильного Junior / Junior+ Python Backend Developer.
